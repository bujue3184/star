# 星际辩台 —— 开发日志 V1.0

> 基于 Three.js + Next.js 的多 AI 模型回合制讨论/辩论平台
> 开发周期：2026-06-03 ~ 2026-06-07
> 开发环境：Node.js v24.15.0, Ollama 0.30.0, RTX 5060 8GB
> GitHub：https://github.com/bujue3184/star

---

## 一、技术栈

| 层 | 技术 | 用途 |
|----|------|------|
| 前端框架 | Next.js 16 (App Router) | 全栈 React 框架 |
| 样式 | Tailwind CSS 4 | 暗色星空主题 UI |
| 3D 渲染 | Three.js + CSS2DRenderer | 星空粒子 + 星球环 + 对话气泡 |
| 数据库 | Prisma 5 + SQLite | 游戏会话/轮次/消息持久化 |
| AI 调用 | Ollama (本地) + REST API (云端) | 5 家厂商统一调用层 |
| 流式传输 | SSE (Server-Sent Events) | 逐 token 实时输出 |
| 包管理 | npm | 依赖管理 |

## 三、核心功能清单

### 3.1 3D 星空场景
- 2500 颗粒子星空（闪烁动画）
- 6 大星球环形排列（AI 形象贴图 + 光晕）
- 中心恒星（裁判席，脉动 + 日冕射线）
- **CSS2D 对话气泡**：每个星球头顶实时显示发言内容
- **恒星气泡**：V4 导演指令显示在恒星上
- OrbitControls 视角控制 + Raycaster 点击交互

### 3.2 AI 模型支持（5 家厂商）
| 前缀 | 厂商 | 示例模型 |
|------|------|---------|
| `ollama/*` | Ollama 本地 | deepseek-r1:8b, qwen:7b, gemma3:4b |
| `openai/*` | OpenAI | GPT-4o, GPT-4o-mini |
| `deepseek/*` | DeepSeek | deepseek-chat, deepseek-reasoner |
| `volcengine/*` | 火山引擎/豆包 | doubao-pro, doubao-lite, doubao-seed |
| `dashscope/*` | 阿里云/千问 | qwen-plus, qwen-max, qwen-turbo |

### 3.3 双模式调度

**自动模式（V4 导演统筹）**：
- DeepSeek V4 动态调度每位选手发言
- 每次只调一人，像真人对话
- V4 根据上下文决定谁发言、说什么方向
- 讨论完成时弹窗让玩家确认结束

**手动模式（上帝模式）**：
- 玩家选择发言者 + 输入指令
- 每个模型独立调用，流式输出
- 适合推理剧本杀、狼人杀等需要私密信息的场景

### 3.4 流式 SSE 输出
- 逐 token 实时推送
- 星球气泡 + 侧边栏同步更新
- 当前发言星球高亮
- 每条发言结束后立即插入对话记录，不等轮次结束

### 3.5 思维链剥离
- deepseek-r1：检测 `...` 标签，过滤思考内容
- qwen3.6/gemma4：检测 `thinking` 字段，优先取 `content`
- 所有推理模型的思维链在流式解析阶段被自动跳过

### 3.6 裁判建议 + 玩家确认
- V4 判定游戏可结束时，不直接终止
- 弹出建议确认框，显示结束原因
- 玩家选择「确认结束」或「继续游戏」
- 确认后 V4 生成最终总结

## 四、开发过程中解决的问题

### 4.1 显存管理（RTX 5060 8GB）
- **问题**：3 个本地模型同时加载 → 13GB > 8GB → OOM
- **解决**：`keep_alive: "0"` + 模型切换间隔 3 秒

### 4.2 SSE 流控制器崩溃
- **问题**：浏览器断连后 `controller.enqueue()` 抛出 "Controller is already closed"
- **解决**：`safeEnqueue()` 捕获并忽略断连错误

### 4.3 思维链泄漏
- **问题**：deepseek-r1 的 `...` 标签、qwen3.6/gemma4 的 `thinking` 字段污染输出
- **解决**：流式过滤 `...` + 检测 `thinking` 字段 + `isChain` 检测器

### 4.4 DeepSeek R1 `</think>` 标签 Bug
- **问题**：`...` 标签在同一 token 中时，`continue` 跳过了后续内容
- **解决**：检测 `</think>` 后提取标签后的内容再 yield

### 4.5 身份混淆
- **问题**：模型看到历史消息中的 `【xxx 发言】` 后模仿该身份
- **解决**：每次发言前重申 `你是【{name}】` + 系统消息过滤

### 4.6 星球闪烁错位
- **问题**：豆包发言时 GPT 星球闪烁
- **解决**：API 路由 `order: bot.order ?? index` 保留前端传入的席位索引

### 4.7 V4 连续调用同一选手
- **问题**：V4 一直 call DeepSeek R1，其他模型没机会说话
- **解决**：统筹规则改为「不能连续两次叫同一个人」

### 4.8 消息即时展示
- **问题**：`bot_done` 用闭包中的旧 `game` 变量找轮次，找不到
- **解决**：`setGame` 函数式更新 + `prev` 保证最新状态

## 五、已知问题 / 待优化

### 5.1 私密消息
- 狼人杀等游戏需要"私密频道"——狼人夜晚的指令不能让预言家看到
- 需要在消息模型加 `visibleTo` 字段

### 5.2 自由讨论
- 多个模型同时发言，谁有想法谁说
- 需要 WebSocket + 模型主动触发

### 5.3 思考过程折叠
- DeepSeek R1 的思维链内容可以折叠展示，用户手动展开查看

### 5.4 大数据模型支持
- qwen3.6:latest（23GB）超出 8GB 显存，跑在 CPU 上极慢
- 需要显存检测和自动降级

## 六、API 接口清单

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/game` | 游戏列表 |
| `POST` | `/api/game` | 创建游戏 |
| `GET` | `/api/game/:id` | 游戏详情（含轮次消息） |
| `DELETE` | `/api/game/:id` | 删除游戏 |
| `POST` | `/api/game/:id/start` | 开始游戏 |
| `POST` | `/api/game/:id/turn` | 非流式下一回合 |
| `POST` | `/api/game/:id/turn/stream` | **流式**下一回合（SSE） |
| `POST` | `/api/game/:id/direct/stream` | **手动指令**（SSE，上帝模式） |
| `POST` | `/api/game/:id/end` | 结束游戏 + 生成总结 |
| `GET` | `/api/models` | 探测可用模型列表 |
| `POST` | `/api/models/test` | 测试模型连通性 |
| `GET/POST` | `/api/skills` | 技能库 CRUD |
| `GET/PUT/DELETE` | `/api/skills/:id` | 单个技能 CRUD |
